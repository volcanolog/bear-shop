Практическая работа №9: Refresh-токены и ротация сессий

Цель: Реализация отказоустойчивой системы авторизации с использованием механизмов обновления токенов (Refresh Token) и защиты от повторного использования (Rotation).

Что добавлено:

    Двухуровневая система токенов:

        Внедрено разделение на Access Token (короткоживущий, 15 мин) и Refresh Token (долгоживущий, 7 дней) для минимизации рисков при краже ключей.

        Реализованы разные секретные ключи для подписи каждого типа токена.

    Механизм ротации (Token Rotation):

        Разработан эндпоинт POST /api/auth/refresh, который при каждом обновлении аннулирует старый refresh-токен и выдает новую пару.

        Реализовано серверное хранилище активных refresh-токенов для возможности мгновенного отзыва всех сессий пользователя.

    Автоматизация на стороне фронтенда (Axios Interceptors):

        Настроен Request Interceptor для автоматической подстановки актуального токена в заголовки всех запросов.

        Реализован Response Interceptor, который незаметно для пользователя обновляет сессию при получении ошибки 401 и повторяет неудавшееся действие.

    Обработка истечения сессии:

        Добавлена логика принудительного логаута и очистки localStorage в случае невалидности или истечения срока действия refresh-токена.

    Тестирование жизненного цикла:

        Проведена проверка сценария «использования старого refresh-токена»: подтверждено, что система блокирует доступ при попытке повторного использования уже обновленного ключа.
